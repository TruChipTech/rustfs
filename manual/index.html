RustFS command manual

RustFS is a multi-call binary: the command name, symlink name, or applet name selects the tool to run. This page gives a multi-call overview of the included applets, their purpose, and the main arguments you are expected to use. It is meant as a quick reference for the current repository, not as a full POSIX man-page replacement.

Syntax note: bracketed items are optional, uppercase names are placeholders, and ... means the item may be repeated.

Getting Started

Use RustFS in either of two ways:

# Multi-call mode
rustfs ls -la /
rustfs grep -n TODO src/

# Symlink mode
ln -s rustfs ls
./ls -la /

In symlink mode, the symlink name decides which applet runs. That is why this project behaves like a standard multi-call utility.

Command Syntax

Flags

Most applet options begin with -. Short flags may be combined when the implementation allows it, for example -cfk.

Arguments

Uppercase words like FILE, HOST, or DEVICE are positional arguments. Fill them with real paths, hostnames, or device names.

Lists

Items ending in ... accept more than one value, such as FILE... or MODULE....

How To Read Arguments

Single dash options

-v usually means verbose, -q quiet, -f force, -r recursive or read-only depending on the command.

Repeated arguments

If the synopsis shows FILE..., pass as many files as needed. The same applies to MODULE..., PID..., and similar groups.

Shell redirection

RustFS shell supports standard redirection forms such as >, >>, <, 2>, 2>>, and 2>&1.

Pipelines

Commands can be chained with pipes, and output redirection on a pipe segment is supported in the shell implementation.
